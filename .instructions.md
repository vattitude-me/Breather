# Breather Project Instructions for Agents

## Project Overview
Breather is a break routine reminder application with PWA and Chrome Extension implementations. The project uses a monorepo structure with shared, PWA, and Chrome extension packages.

## Git Workflow Requirements
1. **Commits**: Automatically commit changes with descriptive messages - no approval needed
2. **Push to Origin**: Always get user approval before pushing to origin
3. **Deletions**: Always get user approval before deleting files or directories
4. **Version Increment**: Automatically increment semantic version (patch) in all package.json files on each commit

## Version Management
When committing changes:
- Increment patch version (e.g., 1.0.0 → 1.0.1) in:
  - `packages/pwa/package.json`
  - `packages/chrome-ext/package.json`
  - `packages/shared/package.json`
- Update version in commit message

## Font & Typography
The PWA has been optimized for clarity and readability with the following standards:
- Font rendering: `subpixel-antialiased` on webkit, `auto` on Firefox
- Base font size: 16px with line-height: 1.5
- UI element font sizes:
  - Navigation items: 12px
  - Labels & chips: 14px
  - Page headers: 24px
  - Subtext: 14px
- Line heights: 1.4-1.5 throughout UI elements
- Letter spacing: 0.8px for labels, 0.3px for buttons

See `/memories/repo/breather-font-improvements.md` for detailed changes.

## Project Structure
- `packages/pwa/` - Progressive Web App implementation
- `packages/chrome-ext/` - Chrome Extension implementation
- `packages/shared/` - Shared utilities and types across packages
- Root level files: Configuration and documentation

## Key Files
- PWA styling: `packages/pwa/src/{index.css,App.css,screens.css}`
- Shared constants: `packages/shared/src/constants.ts`
- Chrome extension manifest: `packages/chrome-ext/public/manifest.json`
